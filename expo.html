What It Does

A profile that deepens with every conversation

Serendipity processes conversations — audio files or pre-written transcripts — and extracts a structured psychological and biographical profile of a person.

The system learns incrementally: every new conversation deepens the profile. Traits seen repeatedly gain higher confidence. Contradictory information gets resolved. Short-term emotional states are replaced with fresh ones.

The project was validated on a real-world dataset: 16 episodes of Friends, pre-transcribed as .txt files. After each episode the graph correctly and progressively captures the character's personality.

🔒 Local-First Philosophy

Everything runs entirely on your machine. No data leaves. No API keys required (except a one-time HuggingFace download). Safe for sensitive personal conversations, deployable fully offline.

📈 Incremental Learning

Confidence starts at 0.6 for new traits. Each re-appearance adds +0.1, capped at 1.0. High-confidence nodes cluster near the center of the 3D graph.

🧠
Deterministic Node IDs
Every trait gets a slug ID — cv_discipline for "discipline" as a core value. Across 16 episodes, the same node accumulates evidence rather than creating duplicates.
💾
Snapshot & Rollback
After every pipeline run, the full graph serializes to a SQLite snapshot. Up to 100 snapshots are retained. A diff view shows exactly what changed — nodes added, strengthened, updated, or removed.
🔄
Smart State Wiping
Short-term emotional states are deleted at the start of each new run. Values and goals accumulate; feelings are always fresh. This mirrors how real personalities work.
🎯
Multi-Pass Extraction
Instead of one broad LLM call, each category (values, goals, states) gets its own focused prompt. A final dedup pass cross-reasons across all categories to find connections a single pass would miss.

Technology Breakdown

Full Stack, Fully Local

Every layer of the system uses open-source, locally-executed tools. No cloud services, no API keys at runtime.

🎙 Transcription Layer
faster-whisper
Local
CTranslate2-based reimplementation of OpenAI's Whisper. Runs fully on CPU (or CUDA). Achieves real-time factor <1.0 — transcribes faster than the audio duration.
silero-VAD
Local
Voice Activity Detection — skips silence, music, and non-speech before Whisper processes anything. Dramatically speeds up transcription on real-world recordings with pauses.
PyAV
Library
Decodes audio to 16 kHz mono float32. Applies peak normalization to −3 dBFS (~70.7% full scale) so Whisper gets consistent input regardless of original recording volume.
👥 Speaker Diarization Layer
pyannote 3.1
Local
State-of-the-art HuggingFace diarization model (~500 MB). Processes audio in 10-min subprocess chunks to stay within RAM limits. Each chunk is killed after completion — memory fully released.
Cosine Similarity
Algorithm
Speaker embeddings from adjacent chunks are compared via cosine similarity. Threshold ≥ 0.65 → same person. Running averages update embeddings over time for better accuracy.
Name Spotting
Algorithm
Two-signal system: self-ID ("I'm Monica", weight 2) and direct address ("Hey Monica", weight 1). Requires total score ≥ 2 and ≥ 40% consistency. 80-word NOT_NAMES blocklist prevents false positives.
🧠 LLM Intelligence Layer
Ollama
Local
Local LLM runtime. Default models: mistral:7b (diarization), llama3 (extraction + gatekeeper). All calls in JSON mode — forces structured output. 180-second HTTP timeout.
Pydantic v2
Library
Validates all LLM outputs. Coercion validators handle: nested label dicts, stringified floats, out-of-range confidence values, null labels. Every coercion increments telemetry counters.
Multi-turn Correction
Algorithm
On malformed JSON: exponential backoff (3 retries, 1s → 2s → 4s + jitter). The bad response is appended to conversation history and a correction prompt is sent — far more effective than blindly resending.
💾 Storage Layer
SQLite + WAL
Database
Zero-config file-based database at data/profiles/<name>/profile.db. WAL (Write-Ahead Log) mode allows concurrent reads during pipeline writes — critical for the live UI.
NetworkX
Library
Directed graph (nx.DiGraph) in memory. One root "user" node. All trait nodes connect via weighted edges (confidence 0–1). Serialized to JSON for SQLite snapshots.
Snapshot System
Feature
Full graph serialized after every run into the snapshots table. Up to 100 retained. diff_graphs() compares any two snapshots for a structured added/updated/removed report.
🌐 UI & Visualization Layer
Streamlit
Framework
Python → interactive web app with minimal boilerplate. 7 tabs: Graph, Profile, Run Days, Stage 1, History, Raw Data, Log. Session state keyed with typed string constants to prevent silent typo bugs.
3d-force-graph
Library
JS library wrapping Three.js + D3-force for interactive 3D graph rendering. Node size ∝ confidence. Edge beam thickness ∝ confidence. Directional animated particles show graph directionality.
Fibonacci Sphere
Algorithm
Nodes placed using golden-angle spiral for uniform surface distribution with no clumping. Radius ∝ 1/confidence: uncertain nodes on outer shell, high-certainty nodes cluster at center.
⚙️ config.py — Environment-Variable Backed Constants

Every tunable number lives in config.py as a module-level constant that reads from an env var first. No code changes needed to tune behavior — just set SERENDIPITY_LLM_TIMEOUT=300 in your shell.
